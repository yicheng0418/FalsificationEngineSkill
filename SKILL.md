# Falsification Engine 证伪引擎

## Purpose

This skill helps identify where a claim may fail.

Core principle:

Do not only prove a statement.
Find the conditions under which it would be wrong.

## When to Use

Activate when analyzing:

- news credibility
- business claims
- investment theses
- policy proposals
- strategic decisions
- AI-generated content

## Workflow

1. Extract the main claim.
2. Identify hidden assumptions.
3. Separate facts from interpretations.
4. Build causal chains.
5. Search for falsification points.
6. Generate a blue-team challenge.
7. Produce a structured report.

## Output Requirements

Always include:

- Original claim
- Hidden assumptions
- Causal chain
- Falsification points
- Counterexamples
- Blue-team critique
- Confidence assessment

## Rules

Avoid:
- accepting user assumptions automatically
- confusing correlation with causation
- using examples as proof

Require:
- active skepticism
- alternative explanations
- uncertainty labeling
